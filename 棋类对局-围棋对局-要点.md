# 围棋对局 · 要点总结

## 棋类对应标准翻译、标识
 围棋  go

 中国象棋 chinese-chess
 国际象棋 chess

> 基于 lan-boardgame 项目，记录围棋对局的关键状态流转和设计要点。

## 一、房间状态

房间有两层状态：**生命周期状态** 和 **活动状态**。

### 1.1 生命周期

```
  创建房间
      │
      ▼
  ┌──────────┐    房主离开/断线超时 ──► 全员弹窗 ──► 返回大厅
  │  ACTIVE  │ ─────────────────────────────────────────► DESTROYED
  └──────────┘
```

- 只要房主在线，房间就活着
- 房主主动离开 → 立即销毁
- 房主断线 → 30 秒保护期，可重连恢复；超时后销毁

### 1.2 活动状态（房主的当前活动）

| 状态 | 含义 | 数据统计 ID |
|------|------|------------|
| `idle_0` | 完全空闲（等人） | 0 |
| `idle_1` | 打谱中 | 1 |
| `idle_2` | AI 对弈中 | 2 |
| `playing` | PvP 对局中 | - |

- 统计用途：`room_logs` 表记录 `idle_type`，后台可分析用户行为分布
- `等对手` = `idle_0`（行为上等同于完全空闲）

## 二、角色与权限

### 2.1 用户在房间中的身份

| 身份 | 图标 | 获取方式 | 说明 |
|------|------|---------|------|
| 👑 **房主** (Owner) | 👑 | 创建房间 | 房间的主人，他离开则房间销毁 |
| 🧑 **玩家** (Player) | 🧑 | 申请对局成功后被拉入 | 对局双方，最多 2 人 |
| 👤 **观众** (Spectator) | 👤 | 加入房间（默认） | 只能看棋和聊天，不能操作棋盘 |

**关键规则**：
- 后进入房间的人**默认都是观众**，不会自动成为玩家
- 只有通过「申请对局 → 房主同意」流程，申请人才从观众变为玩家
- 房主始终在玩家区（👑 前缀），不进入观众区
- **观众区始终显示**，无人时显示「👤 观战 (0) / 暂无观战」

### 2.2 侧边栏操作按钮

| 按钮 | 显示条件 |
|------|---------|
| 🤖 AI 对弈 | 房主 + 无对局中 |
| 📖 打谱 | 房主 + 无对局中 |
| 🏆 申请对局 | 非房主 + 无对局中 |
| 申请人用户名 | 非房主 + 申请对局按钮上方 |
| 同意 / 拒绝 | 房主收到挑战时 |
| ⏭️ Pass | 对局中 + 且是玩家 |
| 🏳️ 认输 | 对局中 + 且是玩家 |
| 🚪 退出房间 | 所有人（按钮在房间标题右侧） |

## 三、申请对局（挑战）流程

```
申请人点击「申请对局」
         │
         ▼
  ┌──────────────────────┐      ┌──────────────────────┐
  │   申请人弹窗          │      │   房主弹窗            │
  │                      │      │                      │
  │   ⏳ (旋转动画)       │      │   ⏳ (旋转动画)       │
  │                      │      │                      │
  │ 已经向 👑xxx 申请对局  │      │ xxx 申请与您对局       │
  │                      │      │                      │
  │    60秒倒计时         │      │    58秒倒计时         │
  │                      │      │                      │
  │ 倒计时结束，自动拒绝！  │      │  ┌────┐ ┌────┐      │
  └──────────────────────┘      │  │同意│ │拒绝│      │
                                │  └────┘ └────┘      │
                                └──────────────────────┘
```

**关键规则**：
- 弹窗**只有申请人**和**房主**能看到，其他人看不到
- ⏳ 动画：2 秒绕一圈，顺时针旋转
- 60 秒倒计时，最后 10 秒数字变红 + 脉冲闪烁
- 房主点「同意」→ 玩家区刷新，对局开始
- 房主点「拒绝」→ 双方弹窗关闭，房主继续原来的空闲活动
  - 申请人收到弹窗提示「👑xxx 拒绝对局」
- 60 秒超时 → 服务端自动取消，双方弹窗关闭
  - 申请人收到提示「申请超时」

## 四、对局流程

### 4.1 落子

- 点击棋盘交叉点 → Canvas 检测坐标 → 发送 `place` → 服务端验证 → 广播 `game_state`
- 白方棋盘自动 180° 反转，己方棋子在下方
- 悬停时显示半透明预览
- 最后一步落子位置有红色标记
- 落子音效：Web Audio API 生成短促「嗒」声

**服务端验证**（`validateMove`）：
1. 游戏是否进行中
2. 是否轮到该玩家
3. 位置是否在棋盘范围内
4. 位置是否为空
5. 是否违反劫规则（禁止立即回提）
6. 是否自杀

### 4.2 Pass

- 点击「⏭️ Pass」→ 切换回合
- 双方连续 Pass → 触发终局计分

### 4.3 认输

- 点击「🏳️ 认输」→ 直接判对方获胜
- 任何回合都可认输，不检查轮次

## 五、终局与再战

### 5.1 终局弹窗内容

| 看到的人 | 内容 |
|---------|------|
| **对局双方** | 胜负文字 + 再战一局 / 退出 + 60 秒倒计时 |
| **观众** | 胜负文字 + 退出（无再战选项） |

### 5.2 胜负文字

| 原因 | 胜方看到 | 败方看到 |
|------|---------|---------|
| 认输 | 对方认输，**你赢了！** | 你认输了！ |
| 断线 | 对方断线，**你赢了！** | 你断线了！ |
| 计分 | ⚫黑棋/⚪白棋**胜利！** | 同上 |

### 5.3 彩带和音效

- **只对胜方播放**：五彩纸屑飘落 + 胜利上升琶音
- 败方和观众不触发

### 5.4 再战一局流程

```
玩家A点「再战一局」
     │
     ├─ A 看到：「已申请再战」（灰）
     └─ B 看到：「对方已申请再战」
              │
          B 点「对方已申请再战」
              │
              ▼
        双方同意 → 刷新棋盘 → 新一局开始！
```

| 状态 | 按钮文字 |
|------|---------|
| 未操作 | 再战一局 |
| 自己已点 | 已申请再战（灰） |
| 对方已点 | 对方已申请再战 |

### 5.5 退出再战

| 操作者 | 效果 |
|-------|------|
| 房主点「退出」 | 棋盘保留终局画面，回到空闲状态 |
| 申请人点「退出」 | 回到观众区，棋盘保留终局画面 |
| 对方已点「退出」 | 按钮变为「对方退出」，弹窗 2 秒后自动关闭 |

退出后：
- 非房主玩家移回观众区
- 房主保留在玩家区
- `room.gameState` 保留（棋盘不清空）
- `activity` 回到 `idle_0`
- 广播 `room_updated` 同步所有人

## 六、棋盘状态流转

```
        空闲（无对局）
         │
         ├── 显示空 19×19 棋盘（淡金色背景）
         │
         ├── 点击「AI 对弈」──► 对局中（黑棋 vs AI 白棋）
         │
         ├── 挑战接受 ──► 对局中（黑棋先手）
         │
         ▼
      对局中
         │
         ├── 落子 ──► `game_state` 广播给所有人
         │
         ├── Pass ──► 切换回合 / 连续 Pass → 终局
         │
         ├── 认输 ──► 终局
         │
         ▼
      终局（棋盘保留）
         │
         ├── 再战一局 ──► 棋盘重置 → 对局中
         ├── 退出 ──► 棋盘保留终局画面
         │
         ▼
      空闲（保留终局棋盘）
```

**关键点**：
- 终局后 `gameState` 保留在服务端，退出再战后也不清空
- `room_updated` 消息中的 `gameState` 字段用于同步棋盘状态
- 客户端收到 `room_updated` 且 `gameState` 为 null 时才会清除棋盘

## 七、断线重连

### 7.1 保护机制

- **房主断线**：30 秒保护期，房间保留在内存但**从大厅列表中隐藏**
- **玩家断线**：保留在房间 30 秒
- **观众断线**：同上

### 7.2 重连流程

```
刷新页面 → WebSocket 重连 → 发送 rejoin_room { roomId, playerId }
    │
    ▼
服务端验证 → 更新 ws 连接 → 取消销毁倒计时
    │
    ▼
推送：完整房间状态 + 聊天历史 + 游戏状态
    │
    ▼
广播 room_updated 给其他人 → 观众列表刷新
```

**恢复的数据**：
- 房间状态（人员列表、活动状态）
- 聊天记录（全部，不限可见性）
- 游戏状态（棋盘、回合、最后落子）

## 八、WebSocket 消息协议

### 8.1 房间管理

| 方向 | type | 说明 |
|------|------|------|
| C→S | `create_room` | 创建房间（需传 config） |
| S→C | `room_created` | 返回 roomId + playerId |
| C→S | `join_room` | 加入房间（默认观众） |
| S→C | `room_joined` | 返回完整房间快照 |
| S→C | `room_updated` | 房间状态变化通知 |
| S→C | `player_joined` | 有人加入 |
| C→S | `leave_room` | 退出房间 |
| S→C | `player_left` | 玩家离开（含断线接管提示） |
| S→C | `room_destroyed` | 房间销毁（所有人回大厅） |
| C→S | `get_rooms` | 获取大厅房间列表 |
| S→C | `room_list` | 返回房间列表 + 统计 |
| C→S | `set_activity` | 房主设置活动状态 |
| S→C | `owner_disconnected` | 房主断线（30秒保护期倒计时） |
| S→C | `player_rejoined` | 玩家重连成功通知 |

### 8.2 游戏

| 方向 | type | 说明 |
|------|------|------|
| C→S | `start_ai_game` | 房主开始 AI 对弈 |
| C→S | `place` | 落子 |
| C→S | `pass` | 虚手 |
| C→S | `resign` | 认输 |
| S→C | `game_started` | 对局开始 |
| S→C | `game_state` | 游戏状态更新 |
| S→C | `game_over` | 对局结束 |

### 8.3 挑战

| 方向 | type | 说明 |
|------|------|------|
| C→S | `challenge` | 申请对局 |
| S→C | `challenge_request` | 通知房主（含申请人用户名） |
| S→C | `challenge_sent` | 通知申请人（已发送） |
| C→S | `challenge_response` | 同意/拒绝 |
| S→C | `challenge_response` | 结果（拒绝时带消息） |
| S→C | `challenge_timeout` | 60 秒超时 |

### 8.4 再战

| 方向 | type | 说明 |
|------|------|------|
| C→S | `rematch` | 申请再战 |
| S→C | `rematch_self` | 自己已申请（仅发给发送者） |
| S→C | `rematch_notify` | 对方申请再战（仅发给对方） |
| C→S | `rematch_response` | 退出再战 |
| S→C | `rematch_exit` | 对方已退出 |

## 九、前端组件结构

```
RoomPage
├── Sidebar（左侧）
│   ├── 房间标题 + 🚪退出按钮
│   ├── PlayerList（玩家区）
│   │   ├── 👑 房主
│   │   └── 🧑 其他玩家（对局中显示⚫⚪）
│   ├── SpectatorList（观战区）← 始终显示
│   │   ├── 👤 观战 (N)
│   │   └── 暂无观战（空时）
│   └── SidebarActions（底部按钮区）
│       ├── AI 对弈 / 打谱（房主）
│       ├── 申请对局（非房主）
│       ├── 同意/拒绝（房主被挑战时）
│       └── Pass / 认输（对局中玩家）
│
├── GoBoard（中间 - Canvas 棋盘）
│   ├── 网格 + 星位 + 坐标
│   ├── 棋子渐变绘制
│   ├── 最后一步标记
│   └── 悬停预览
│
├── ChatPanel（右侧）
│   ├── 消息列表
│   └── 输入框
│
└── Modals（弹窗）
    ├── 挑战弹窗（⏳ + 倒计时）
    ├── 终局弹窗（胜负 + 再战/退出）
    ├── 确认弹窗（通用）
    └── 提示弹窗（通用）
```

```sql
id, game_type, rule_set, board_size, players (JSON),
```

```sql
room_id, owner_id, owner_name, game_type, config (JSON),
activity, player_ids (JSON), created_at
```

```sql
id, room_id, owner_id, game_type, player_count, idle_type,
```
