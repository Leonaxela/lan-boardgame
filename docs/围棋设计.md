# 围棋实现设计

## 概述

作为首发游戏，在 `games/go/` 下实现完整的围棋游戏，包括两套规则引擎（中国规则 + 日韩规则）、AI、棋盘组件和房间页面。

## 棋盘与规则

- **棋盘**: 19×19（默认）/ 13×13 / 9×9
- **黑方先行**（有让子时白先）
- **棋子**: 黑方（⚫）/ 白方（⚪）
- **猜先**: 猜单双，猜对者执黑

### 基本规则

| 规则 | 说明 |
|------|------|
| 落子 | 棋子放在交叉点上，落子后不能移动 |
| 气 | 棋子上下左右相邻的空交叉点 |
| 提子 | 某群棋子无气时被提走 |
| 禁止自杀 | 落子后己方无气且未提掉对方则禁止 |
| 劫 | 刚被提子的位置禁止立即回提 |
| 虚手（Pass） | 可以选择不下子，轮到对方 |
| 终局 | 双方连续虚手时终局 |

### 两套规则

| 规则 | 计分方式 | 贴目 | 终局判定 |
|------|---------|------|---------|
| 中国规则（数子制） | 数盘上活子数 | 3¾ 子（7.5 目） | 黑子数 vs 白子数 + 贴目 |
| 日韩规则（数目制） | 数目 + 死子 | 6.5 目 | 领地数 + 提子 - 贴目 |

### 让子规则

- 让子数: 0-9 颗
- 让子放置在标准星位上
- 有让子时白方先行

## AI 设计

围棋 AI 采用三档不同算法，难度越高算法越强：

| 难度 | 算法 | 大约人类水平 |
|------|------|------------|
| 简单 | 基础启发式评分（提子/领地/位置） | ~10-15 级 |
| 普通 | 完整启发式评分（提子/逃子/打吃/领地/压制/位置/开局/填眼/劫） | ~业余 1-2 段 |
| 困难 | MCTS 蒙特卡洛树搜索（800 次迭代 + UCB1 + 领地估算 rollout） | ~业余 3-5 段 |

### AI 难度与人类水平对照

| 难度 | 算法 | 大约段级位 | 人类水平 |
|------|------|----------|---------|
| 简单 | 启发式评分 | 10-15 级 | 入门级 |
| 普通 | 完整启发式 | 业余 1-2 段 | 中级棋手 |
| 困难 | MCTS | 业余 3-5 段 | 强业余棋手 |

## 棋谱保存

- **先手**: 黑方（有让子时白先）
- **存储位置**: `game_records` 表的 `sgf` 字段
- **棋谱格式**: SGF（Smart Game Format）
- **格式说明**: 标准围棋棋谱格式，使用坐标字母 a-s + 行号
- **示例**: `(;GM[1]FF[4]SZ[19]KM[7.5]RU[chinese];B[pd];W[dp];B[pp])`




## KataGo 设计

### 概述

KataGo 是目前最强的开源围棋 AI，基于蒙特卡洛树搜索（MCTS）+ 深度神经网络。项目集成 KataGo 作为可选的高级 AI 对手。

### 技术架构

```
server/src/katago/
├── KataGoManager.ts          # 管理 KataGo 子进程生命周期
├── gtpCoordinates.ts         # GTP 坐标 ↔ 内部坐标互转
└── katago-v1.16.5-opencl-windows-x64/  # KataGo 可执行文件
    ├── katago.exe
    ├── default_gtp.cfg
    └── KataGoData/
```

### 通信协议

- **协议**: GTP（Go Text Protocol）
- **通信方式**: stdin/stdout
- **每间房一个独立进程**: 通过 `roomId` 隔离

### 支持的棋盘大小

| 棋盘 | 支持 |
|------|------|
| 19×19 | ✅ |
| 13×13 | ✅ |
| 9×9 | ✅ |

### 难度设置

通过 `maxVisits`（最大搜索次数）控制难度：

| 难度 | maxVisits | 说明 |
|------|-----------|------|
| 简单 | 500 | 搜索较浅 |
| 普通 | 1000 | 中等搜索 |
| 困难 | 2000 | 搜索更深 |

### 每步思考时间限制

KataGo 每步思考受两个限制，**取先到的**：

| 限制 | 值 | 说明 |
|------|-----|------|
| maxVisits | 500/1000/2000 | 搜索次数上限 |
| maxTime | 60秒 | 每步最大思考时间 |

启动参数：`-override-config maxVisits=${visits},maxTime=60`

服务端保护超时：150 秒（防止 KataGo 卡死）

### 模型信息

- **模型**: kata1-zhizi-b28c512nbt-muonfd2
- **架构**: b28c512nbt（28层残差网络，512通道）
- **来源**: KataGo 分布式训练 + ZhiziGo 优化
