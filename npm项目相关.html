npm 项目相关知识

一、npm workspace 模式

当我们根目录的 package.json 中写了 "workspaces" 字段，npm 就进入了 workspace 模式（也叫 monorepo 模式）。

1.1 什么是 workspace？

一个项目不是单独的包，而是由多个包组成。每个子目录都是一个独立的 npm 包，有各自的 package.json。

以本项目为例：

// 根目录 package.json
{
  "name": "lan-boardgame",
  "private": true,
  "workspaces": [
    "shared",
    "client",
    "server",
    "games/*"
  ]
}

1.2 本项目的包结构

lan-boardgame/ ← 根项目（workspace 入口） ├── package.json ├── package-lock.json ├── node_modules/ ← 所有包共享这一份依赖 │ └── @lan-boardgame/ │ ├── shared/ ──────────→ shared/ │ ├── client/ ──────────→ client/ │ ├── server/ ──────────→ server/ │ ├── go/ ──────────→ games/go/ │ ├── gomoku/ ──────────→ games/gomoku/ │ ├── chess/ ──────────→ games/chess/ │ └── chinese-chess/ ──→ games/chinese-chess/ ├── shared/package.json ← name: "@lan-boardgame/shared" ├── client/package.json ← name: "@lan-boardgame/client" ├── server/package.json ← name: "@lan-boardgame/server" └── games/ ├── go/package.json ← name: "@lan-boardgame/go" ├── gomoku/package.json ← name: "@lan-boardgame/gomoku" ├── chess/package.json ← name: "@lan-boardgame/chess" ├── chinese-chess/package.json └── draughts/package.json ← name: "@lan-boardgame/draughts"

1.3 workspace 的好处

  • 依赖共享：所有包的依赖（express、ws 等）只在根 node_modules 里装一份，不重复
  • 互相引用：子包之间可以直接用 @lan-boardgame/xxx 互相 import，不需要发布到 npm
  • 统一管理：在根目录执行 npm install 一次，所有包的依赖都会安装

二、符号链接（Symlink）

2.1 什么是符号链接？

符号链接就是一个快捷方式 / 指针，指向实际的文件或目录。它本身不包含内容，只是告诉系统："去找那个目录"。

2.2 workspace 中的符号链接

执行 npm install 后，npm 会在根目录 node_modules/@lan-boardgame/ 下为每个 workspace 包创建一个符号链接：

node_modules/@lan-boardgame/chess ──→ games/chess/ node_modules/@lan-boardgame/go ──→ games/go/ node_modules/@lan-boardgame/gomoku ──→ games/gomoku/

这样，当 server 里写：

// server/src/websocket/Dispatcher.ts
import { ChessEngine } from '@lan-boardgame/chess/engine';

Node.js 的解析过程是：

  1. 找到 node_modules/@lan-boardgame/chess（符号链接）
  2. 顺着链接找到 games/chess/（实际目录）
  3. 根据 package.json 的 "exports" 字段，定位到 src/rules/engine.ts

2.3 符号链接缺失会怎样？

如果 node_modules/@lan-boardgame/draughts 不存在，Node.js 会在 node_modules 里找不到 @lan-boardgame/draughts 这个包，抛出 ERR_MODULE_NOT_FOUND 错误。

三、package-lock.json

3.1 什么是 lock 文件？

package-lock.json 是 npm 自动生成的依赖清单，记录了项目所有依赖包的：

  • 精确版本号（比如 ^4.21.1 可能安装了 4.21.2）
  • 下载地址（resolved URL）
  • 校验码（integrity hash，确保包内容没被篡改）
  • 依赖树（每个包的子依赖是什么）

3.2 workspace 模式下的 lock 文件

在 workspace 模式下，lock 文件还会记录哪些本地包被链接进了 node_modules。npm 通过 lock 文件判断需要创建哪些符号链接。

你的 lock 文件里记录了 chess、go、gomoku 等，但没有 draughts ——因为上次 npm install 时 draughts 还不存在。

3.3 为什么不能手动编辑？

lock 文件除了包名外，还包含 resolved URL、integrity hash、嵌套依赖树等复杂信息，手动编写容易出错且格式难以维护。

四、本次问题排查实战

4.1 错误现象

// 执行 npm run server 后报错：
Error [ERR_MODULE_NOT_FOUND]: Cannot find package '@lan-boardgame/draughts'
  imported from D:\...\server\src\websocket\Dispatcher.ts

4.2 排查过程

  1. 检查 games/draughts/package.json → 存在，配置正确
  2. 检查 node_modules/@lan-boardgame/ 目录 → 没有 draughts 的符号链接
  3. 检查 package-lock.json → 没有 draughts 的记录
  4. 对比其他游戏包（chess、gomoku 等）的 package.json → 配置完全一致

4.3 根因

draughts 是在上次 npm install 之后才创建的，所以 lock 文件和符号链接中都没有它。

4.4 修复方法

在项目根目录重新执行一次 npm install，npm 会重新扫描所有 workspace 包，为 draughts 创建符号链接并更新 lock 文件。

5.1 为什么 Emoji 最先输出？

关键在于 import 是在文件顶部声明的，Node.js 加载模块时会先解析所有 import，再执行代码。

加载链路：

// index.ts 第 7 行
import { GameWSServer } from './websocket/WSServer.js';

// WSServer.ts 第 12 行
import { handleEmojiMessage } from '../emoji/EmojiGameManager.js';

// EmojiGameManager.ts 第 48 行（模块顶层，非函数内）
loadQuestions();  // ← 加载完就立即执行，输出日志

所以：index.ts import 了 WSServer.ts，WSServer.ts import 了 EmojiGameManager.ts，而 EmojiGameManager.ts 在模块顶层直接调用了 loadQuestions()，不需要等 start() 执行。

5.2 两条 DB 日志的间隔

// connection.ts
export async function initDb() {
  // ... 初始化 SQL 引擎 ...

  if (existsSync(DB_PATH)) {
    db = new SQL.Database(buffer);
    console.log('[DB] 已加载数据库文件');   // ← 第 2 条日志
  }

  // ... 执行 schema.sql、迁移、清空过期数据 ...

  console.log('[DB] 数据库已初始化');        // ← 第 3 条日志
}

两条日志之间的间隔是数据库初始化耗时（读取 schema、执行 SQL、迁移等）。

5.3 两条 Server 日志

// index.ts
httpServer.listen(PORT, () => {
  console.log(`[Server] http://localhost:${PORT}`);   // ← 第 4 条
  console.log(`[Server] WS: ws://localhost:${PORT}`); // ← 第 5 条
});

这两条在 listen 的回调函数中，表示 TCP 端口监听成功。

六、常用 npm workspace 命令

  • npm install — 在根目录执行，安装所有依赖并创建符号链接
  • npm install <pkg> -w @lan-boardgame/server — 给指定子包安装依赖
  • npm run dev -w server — 在指定子包中运行脚本
  • npm ls --all — 查看完整的依赖树
